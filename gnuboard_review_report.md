# 삼성더클성장의원 그누보드5 이식 프로젝트 종합 코드 리뷰 보고서

본 보고서는 **소아 내분비 전문 클리닉(삼성더클성장의원)** 홈페이지의 그누보드5(Gnuboard5) 이식용 테마(`the_cl`) 및 전체 소스 코드의 아키텍처, 구현 방식, 사용 기술을 심층 분석하고 리뷰한 결과입니다.

---

## 1. 프로젝트 개요

*   **사이트명**: 삼성더클성장의원 (Samsung The CL Growth Clinic)
*   **분야**: 소아 내분비 / 성장 클리닉
*   **개발 형태**: 반응형 원페이지 메인 + 다중 서브페이지 구성의 그누보드5 테마 형태
*   **사용 언어**: PHP, HTML5, CSS3, Vanilla JavaScript
*   **의존 라이브러리**: jQuery (그누보드 기본), Lenis (부드러운 스크롤)

## 3. 페이지 구성 및 공통 레이아웃

### 3-1. 공통 include 구조와 테마 경로화
그누보드의 독립된 테마로 작동할 수 있도록 공통 레이아웃 분리와 경로 유연성을 확보했습니다.
*   **경로 변수 `$G5_URL` 바인딩**: `head.php`에서 `$G5_URL = G5_THEME_URL;`로 정의하여 이미지, 스타일시트, JS 등 테마 하위 자산을 절대경로 형태로 불러올 수 있도록 구현했습니다.
*   **레이아웃 템플릿 분리**: 시각적인 요소가 담긴 `header.html.php`와 `footer.html.php`를 별도 분리하여, 그누보드 자체 BBS 페이지(로그인, 회원가입 등)에서도 동일한 헤더/푸터 마크업을 손쉽게 로드하여 일관된 디자인 유지가 가능합니다.

### 3-2. 서브페이지 매핑
*   **About 그룹**: 병원의 철학, 의료진 소개 및 상세 약력, 원장 인사말, 인테리어 전경, 약도를 안내합니다.
*   **Clinic 그룹**: 성장 장애 및 내분비 질환들에 대한 의학 정보 페이지들로 구성되어 있으며, 세부 과목에 최적화된 마크업으로 가독성을 높였습니다.
*   **Info 그룹**: 진료 일정, 절차, 정밀 검사 항목 소개 및 의료법 고시용 비급여 진료비 내역표를 깔끔한 테이블 레이아웃으로 담고 있습니다.

## 4. 게시판 구현 현황 및 데이터 연동

프로젝트 내부의 커뮤니티 성격 페이지들은 그누보드5의 데이터베이스(MySQL)와 유기적으로 결합되어 실시간 데이터를 바인딩합니다.

### 4-1. 공지사항 게시판 (`sub/sub4_3.php`, `sub/sub4_3_view.php`)
*   **데이터 테이블**: `g5_write_notice` 테이블과 연동됩니다.
*   **동작 방식**: 
    1. `g5_board` 테이블에서 `bo_table = 'notice'`에 해당하는 게시판 설정을 가져와 공지글 번호 목록(`bo_notice`)을 파악합니다.
    2. 상단에 공지 설정된 글(`wr_id` 매칭)을 먼저 정렬하여 출력합니다.
    3. 일반 게시물들은 공지글을 제외한 뒤 페이징 규칙(`LIMIT $from_record, $page_rows`)을 적용하여 쿼리합니다.
    4. 작성 날짜 포맷팅 및 조회수(`wr_hit`), 댓글 수(`wr_comment`), 24시간 이내 새 글 아이콘(`N`) 표시가 완벽하게 PHP 코드로 구현되어 있습니다.

### 4-2. 치료 사례 게시판 (`sub/sub4_1.php`, `sub/sub4_1_view.php`)
*   **데이터 테이블**: `g5_write_gallery` 및 `g5_board_file` 테이블과 조인(Join) 연동됩니다.
*   **동작 방식**:
    1. 사례 페이지 특성상 갤러리 형태로 출력하기 위해 `g5_board_file` 테이블에서 첨부파일(`bf_no = 0`) 정보를 `LEFT JOIN`하여 썸네일 경로를 빌드합니다.
    2. 업로드된 대표 이미지가 없는 경우 테마 내 기본 폴백(Fallback) 이미지(`programs.png`)를 출력하도록 안전망이 구축되어 있습니다.
    3. 본인인증 및 권한 통제를 염두에 두어 `$is_member`가 참일 경우에만 상세 화면 진입을 허용하거나 글쓰기 폼 버튼을 노출하도록 설계되었습니다.

### 4-3. 과도기적 더미 데이터 (`sub/_board_data.php`)
*   이식 전 로컬 정적 PHP 서버 테스트용이나 데이터 미구축 시의 폴백 처리를 위해 공지사항(`$board_notice`) 및 치료 사례(`$board_case`) 배열 데이터를 관리하는 더미 파일이 설계되어 있습니다. 향후 그누보드 DB 테이블 연동이 비활성화되거나 테스트 환경일 때 디버깅용으로 쓰기 적합합니다.

## 5. 회원 관련 스킨 및 인증 처리

`skin/member/basic/` 경로에 그누보드 표준 회원 폼의 필드명(Name attributes)과 액션 경로를 완벽하게 유지하면서 테마 고유의 스타일을 입힌 회원 스킨이 제작되었습니다.

*   **로그인 스킨 (`login.skin.php`)**:
    *   그누보드 전역 액션 경로 `$login_action_url`을 폼의 action으로 지정했습니다.
    *   아이디(`mb_id`), 비밀번호(`mb_password`), 자동 로그인(`auto_login`), 리다이렉트 URL(`url`) 정보를 전송합니다.
    *   자동 로그인 체크 시 공공장소 정보 유출 경고 컨펌창 스크립트를 포함하고 있습니다.
*   **회원가입 폼 (`register_form.skin.php`)**:
    *   약관 동의 단계(`register.skin.php`)를 거쳐 도달하며, 닉네임, 이메일, 생년월일, 연락처 등 기본 그누보드 회원가입 쿼리에 필요한 구조를 유지합니다.
*   **스타일 일관성**:
    *   기존 그누보드의 지저분한 테이블 스타일을 배제하고, 테마의 핵심 클래스인 `auth-card`, `form-input`, `btn-submit`을 사용하여 모던한 반응형 로그인/가입 카드를 구현했습니다.

## 6. 스타일(CSS) 및 디자인 시스템

스타일시트(`css/style.css`)는 고유한 비주얼 정체성을 제공하기 위한 프리미엄 스타일 기법들이 집약되어 있습니다.

### 6-1. 컬러 및 서체 변수화 (CSS Variables)
*   **메인 테마 컬러**: 생명력과 성장을 상징하는 Vivid Green 계열(`--c-primary: #2E8B57`, `--c-primary-light: #3CB371`, `--c-primary-dark: #1B6B3A`)을 주조색으로 사용했습니다.
*   **포인트 컬러**: 고급스러움을 극대화하기 위해 Warm Gold 계열(`--c-accent: #B8963E`, `--c-accent-light: #D4B96A`)을 적용했습니다.
*   **서체 조합**: 장식적이고 기품 있는 영문 세리프 폰트 `Cormorant`와 현대적인 한글 고딕 폰트 `Noto Sans KR`을 유기적으로 배치했습니다.

### 6-2. 비주얼 아트워크 및 UX 디테일
*   **오로라 물결 배경 (Aurora Background)**: 24px 간격의 도트 패턴과 두 종류의 은은한 원형 그라데이션(Sage Green & Gold)을 4중 중첩하고, `@keyframes aurora-move` 애니메이션을 사용해 **32초 주기로 배경이 서서히 일렁이는 고급 비주얼**을 연출했습니다.
*   **텍스트 처리**: 모바일 등 좁은 화면에서 영문이나 한글 단어가 부자연스럽게 중간에 쪼개지는 현상을 방지하기 위해 `word-break: keep-all;`과 `overflow-wrap: break-word;`를 레이아웃 전체에 걸쳐 선언했습니다.
*   **불법 불펌 차단**: 병원 사이트의 신뢰도 저해 및 무단 불펌을 막기 위해 `body { user-select: none; }`를 지정하고, 입력 영역(`input`, `textarea`)에서만 드래그 및 선택이 가능하도록 선별적 폼 드래그를 허용했습니다.

## 7. 스크립트(JS) 및 인터랙션 기술

자바스크립트(`js/main.js`)는 현대 웹 프론트엔드의 최적화 기법을 담아 부드러우면서도 리소스를 적게 소모하도록 설계되었습니다.

### 7-1. Lenis Smooth Scroll 제어
*   기본 브라우저의 딱딱한 스크롤 환경 대신 관성 스크롤을 구현하는 **Lenis 라이브러리**를 연동했습니다.
*   `lerp: 0.15` (기민한 반응성), `wheelMultiplier: 1.2` (마우스 휠 감도 보완), `duration: 0.8` 옵션을 부여하여 휠 동작 시 이질감 없이 스무스하게 페이지가 넘어가도록 튜닝했습니다.
*   `a[href^="#"]` 형식의 해시 링크 클릭 시, 상단 고정 헤더 높이만큼을 자동으로 뺀 계산 위치로 스크롤을 제어해 정밀한 이동을 돕습니다.

### 7-2. Intersection Observer 기반 Scroll Reveal
*   화면을 아래로 내릴 때 콘텐츠 카드가 자연스럽게 위로 올라오는 효과(`.reveal`)를 구현했습니다.
*   스크롤 이벤트 리스너에서 위치를 계산하는 기존 방식 대신, 브라우저가 화면 진입율을 비동기식으로 관찰하는 **Intersection Observer API**를 도입했습니다.
*   화면 하단 15% 지점에 도달할 때 1회만 애니메이션 클래스(`.revealed`)를 부여하고 감시를 바로 해제(`observer.unobserve(entry.target)`)함으로써 불필요한 GPU 렌더링 비용을 원천 차단했습니다.

### 7-3. 포인터 이벤트 기반 가속 모바일 오버레이 메뉴 (핵심 기술)
*   모바일 환경에서 햄버거 메뉴를 클릭할 때 나오는 꽉 찬 화면 오버레이에 고성능 가상 스크롤을 탑재했습니다.
*   스크롤바 영역을 화면에서 제거한 후, 터치 드래그 및 마우스 드래그를 모드 구분 없이 처리하도록 **Pointer Events**로 휠/드래그 이벤트를 바인딩했습니다.
*   휠 및 드래그 변동폭에 프레임 단위 보간 스무딩(`lerp: 0.15`)을 적용하여 CSS `transform: translate3d(0, -Ypx, 0)` 속성으로 미끄러지듯 이동시킴으로써 네이티브 모바일 앱과 같은 메뉴 조작감을 선사합니다.
*   드래그 직후 의도치 않게 메뉴 링크가 클릭되는 문제를 차단하기 위한 Pointer 캡처 방지 스크립트까지 결합되어 완성도가 매우 높습니다.

## 8. 종합 코드 리뷰 의견 및 이식 완료도

### 8-1. 우수한 디자인 및 기술적 성취
*   **미적 완성도**: 세련된 컬러셋과 미세한 오로라 배경 애니메이션이 결합되어 단순 정보 제공형 병원 사이트를 넘어선 럭셔리 클리닉 느낌을 잘 전달합니다.
*   **최적화**: 모바일 가상 스크롤, 비동기 옵저버 기반 모션 reveal 등 성능 누수를 철저히 통제한 코딩 방식이 돋보입니다.
*   **그누보드 친화성**: 변수 구조와 로그인/가입 스킨 폼 필드들을 그누보드 사양과 100% 일치시켜 제작하여 이식 작업에 따른 부작용을 최소화했습니다.

### 8-2. 최종 런칭을 위한 개선 권고사항 (To-Do)
1.  **게시판 명칭 동기화**: `sub4_1.php`에서 사용한 갤러리 테이블(`gallery`)과 `sub4_3.php`에서 사용한 공지사항 테이블(`notice`)이 실제 그누보드 백엔드 어드민에 생성된 게시판 ID(테이블명)와 일치하는지 재확인 및 변경해야 합니다.
2.  **보안 문자(Captcha) 스킨 정비**: 회원가입 폼 등에서 `kcaptcha` 등 그누보드 고유의 보안 방지 라이브러리와 템플릿 마크업이 조화롭게 출력되는지 스킨 파일을 검증할 필요가 있습니다.
3.  **외부 에셋(CDN) 로컬화**: 현재 Google Fonts 및 Lenis 관성 스크롤 라이브러리 등이 CDN 링크로 연결되어 있습니다. 의료기관 사이트의 안정적인 무중단 서비스 및 로딩 속도 극대화를 위해 향후 소스 코드 로컬 폴더로 다운로드받아 번들링하는 것을 추천합니다.
